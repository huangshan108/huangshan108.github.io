Friday, June 6 2014

Today I mainly worked on some validation tasks. I fixed some bug when creating/changing passwords. Schoolmint used to give an error page when user creating/changing a invalid password (not enough length/ a password with all spaces in it… etc...). I fixed it by update the validator scheme and add custom validator to password creating/changing process and show inline warning messages to user when they use an invalid password rather than giving them an error page. I also update the phone number validator so that people much input phone number of 10 digits because all the American phone numbers are 10 digits or 7 without area code. To avoid people mistype their phone number in fewer digits and it is not quite make sense to enter a phone number without an area code here, I implemented and gave them a inline text warning to force them enter a phone number strictly in 10 digits. I also fix a UI problem which causes some long text to overflow the text boxes by hidden the longer part and show the full text when hovering mouse over it. Besides these, I also correct some typos and improved some coding styles.

  • For new sign up user: we do a browser language detection and set their preferred they change it.
  • For a returning sign in user: we get their preferred language in our database and display the webpage in their preferred language. The language of sign in page will be determined by browsers’ language.
  • Currently ONLY supports Spanish, everything else will be treated as English which is by default.
  • User can change their language preference any time by switching languages.
